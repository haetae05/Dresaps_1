온보딩
- 카메라 동의부터 구함
- 동의 구하면 카메라 화면으로 넘어감
- 화면 상단엔 가이드라인 나옴
- 촬영 버튼 없음
- 유저의 몸 위에 실시간 녹색 와이어프레임이 덧씌워짐
- 관절 포인트에 타겟 마커가 생성되어 올바른지 알림
- 스캔에 대한 내용
  1. 가이드라인에 맞춰 3초간 정지 시, 화면에 체크 표시가 역동적으로 가운데에서 생성되어 살짝 올라왔다가 다시 내려가며 사라짐
  2. 너무 어두우면, 가이드라인에 공간이 어두워 온보딩이 어렵다는 가이드라인 출력
  3. 전신이 보여야하기에 신체 일부가 잘리거나 너무 가까우면 뒤로 물러나달라는 가이드라인 출력
  4. 총 4회 촬영 (앞 좌 우 뒤)
  5. 촬영 성공하면 오른쪽으로 돌라고 시킨 다음 촬영을 함.
  6. 가이드라인에 맞지 않는 면(앞 찍어야하는데 옆을 함 등등)을 하면 촬영 거부하며 가이드라인에 가이드 출력
  7. 전신 촬영 후, 얼굴 스캔을 시작함
  8. 유저의 얼굴의 포인트들을 마킹하거나 해서 유저의 얼굴 모양에 대한 모든 정보를 땀
  9. 스캔 과정에서 유저의 피부색, 퍼스널컬러, 동공 모양을 분석하고 스캔해서 기억함.
  10. 전신 촬영 시, 유저의 관절을 큰 거부터 마킹해서 따고 그 다음으로 작은 관절들 순서로 해서 어느정도는 관절의 모양을 땀.
  11. 유저의 관절 뿐 아니라, 몸의 형태와 비율, 크기, 너비 등을 전부 기록하여 아바타 생성에 사용
  12. 스캔에 사용한 모든 이미지는 스캔 이후 영구 삭제, 수치와 기록 데이터만 남음
  13. 스캔에서 색을 따내는 건 피부색이 유일함. 동공 색이라든가 이런 것은 따지 않음.
  14. 머리카락, 손톱, 문신과 같이 언제든지 쉽게 바꾸는 게 가능한 것들(털이라든가 렌즈도 있고)은 스캔하지 않음

**아바타 생성**
- 아바타는 유저의 비율, 크기, 너비, 관절, 체형, 피부색, 동공 모양 등을 전부 기록했던 것들을 바탕으로 생성
- 회원가입할 때 땄던 성별로 아바타 선택(여자 아바타냐 남자 아바타냐)
- 얼굴 생성은 여기서 안 함(AI 이미지 생성할 때 함)
- 즉, 색 관련은 유일하게 피부톤만 존재
- 여기까지 생성을 해놨다가 유저가 키랑 몸무게를 입력하면 그때 그 정보에 맞춰 마감 작업
- 이것은 유저 개인의 아바타가 됨
- 새로 온보딩 하면 이전 아바타는 바로 폐기


홈

포즈 생성
- 포즈 생성 화면 전에 확인하는 단계 진입
- 포즈 티켓이랑 필름 재화량이 우측 상단에 표기됨
- 포즈 티켓 1개 or 필름 20개 있어야만 실행가능
- 포즈 티켓 있으면 포즈 생성 클릭. (만약 필름만 있으면 필름 20개 차감)
- 포즈 생성 화면 진입 (여기서 하기 싫으면 좌측 상단 나가기 버튼. 버튼 누르면 포즈 생성 취소하시겠습니까 알림 뜨고 확인이랑 취소 중 확인 누르면 나가지면서 필름 20개로 환불)
- 포즈 생성은 유저의 관절 모양으로 땀
- 포즈 생성할 때 먼저 유저의 전신을 스캔하는데, 만약 너무 유저가 아닌 것 같으면 (체형이 너무 어긋남) 거부.
- 유저가 카운트를 설정하고 촬영 버튼을 누르면 카운트가 시작되고 카운트가 멈추는 타이밍에 존재하는 유저의 포즈의 관절들의 지점을 찍고 어떻게 어떻게 해서 포즈를 따냄.
- 포즈를 다 따면 잠시 생성 중이라는 화면으로 넘어가고 다 생성됐다고 뜨고나서 포즈명 지을 수 있음

- **Detail**:
  - **Joints**: 어깨, 팔꿈치, 무릎 등 주요 관절.
  - **Extremities**: 손가락(Finger), 발가락(Toe) 끝까지 정밀 추적.
  - **Head**: 고개 젖힘(Tilt), 회전각(Rotation) 포함.



<리코딩>
리코딩 전 선택 페이지.
- **'생성 페이지(New Activity)'** 뜸 (집중도 향상).
- **Auto Load**: 진입 시 유저의 **'온보딩 아바타(Base Body)'**가 중앙에 자동
  로드됨.
유저는 **단 한 번의 세션**으로 최대 **3개의 서로 다른 스타일**을 생성할 수
있습니다.
- **Structure**: 아바타 밑에 `[Slot 1]`, `[Slot 2]`, `[Slot 3]` 탭 제공.
- **Per Slot Settings**:
  1. **Pose**:  보유한 포즈 선택.(기존에 주어지는 포즈 22종)
  2. **Mask**:  가면 선택 (가면 쓰기 싫으면 None 선택. 기존에 주어지는 가면 3종 <- 얘네는 색 선택 가능)
  3. **Background**: 분위기에 맞는 배경 선택.(배경은 일단 기본적으로 주어지는 배경 10종)
- **Credit Logic**: 생성 완료 시, 활성화된 **'슬롯의 개수'만큼** 일일 생성
  횟수(Film)가 즉시 차감됩니다.
- 적용할 때마다 위쪽의 온보딩 아바타가 있는 공간에 해당사항이 실시간 적용됩니다.
- 3개의 슬롯은 각각 전부 다른 것을 선택 가능합니다.
- 리코딩하기 버튼 누르면 촬영으로 넘어갑니다.

리코딩하는 촬영 페이지
- 유저의 체형 아바타를 기반으로, 그 외의 것들은 전부 아이템 취급합니다. (머리카락, 동공 색, 손톱, 가방, 모자, 옷 전부)
- 온보딩과 비슷한 과정을 거칩니다. (얼굴 촬영은 뺍니다.)
- 이미지 스캔이 끝나면 자동으로 생성 로딩 화면으로 넘어갑니다.

결과물 생성 로딩
- **UI**: 기계적인 로딩 바 대신 유저의 마음을 대변하는 문구 출력.
- "제발 예쁘게 나오기를...", "옷 핏을 맞추고 있어요...", "인생샷 건지는 중...". "오늘 좀 괜찮네?" "옷 꽤나 입는구나" 같은 메세지 출력으로 로딩 지루함 해소


결과물 생성
- 아이템 중에 상의, 하의, 신발에 대한 설계도만 따로 땁니다.(게시물 쪽에서 쓸려고 따로 생성하세요)
- 아이템 전체(상의, 하의, 신발 포함임)를 하나로 취급으로 하여서 4면을 찍은 것으로 정확한 아이템들의 설계도를 따고 거기에 4면 찍은 걸로 얻은 색을 집어넣고 질감도 같이 집어넣어 설계도를 완성합니다.
- 완성한 설계도를 바탕으로 아바타 + 포즈 + 가면 + 배경 + 유저의 얼굴 정보 + 설계도 결과물로 재료 준비를 마칩니다.
- AI로 재료들을 조합해서 이미지를 생성합니다. (이 과정에서 정말 정확하고 자세하고 실패없는 결과물을 위해 노력해야합니다.)
- AI 이미지는 3:4입니다.

결과물 생성 완료 및 이미지 편집
- 여기부터 임시 저장 가능합니다. (임시 저장한 결과는 옷장에 있는 결과물 있는 곳으로 갑니다)
- **Zoom**: 핀치 줌으로 최적의 구도(Crop) 설정.
- **Filter**: 색감/분위기 보정만 허용 (형태 왜곡/성형 불가).
- **Text**: 사진 위에 텍스트 배치 (블로그/인스타 스토리 스타일).
- 완료 시 이름 짓기로 넘어갑니다

룩 이름 짓기 및 저장
- **Naming**: "이 룩(Look)의 이름을 지어주세요" (예: 2/4 벚꽃 데이트).
- **Archiving**: 저장 시 옷장으로 이동하며 리코딩 과정 종료 (비공개 상태).

게시물 올리기
- 옷장으로 들어가서 만들어둔 룩을 선택합니다.
- 룩이
- 게시물 생성에는 다음 내용들이 뜹니다.
  1. 게시물 제목(여기도 태그 됩니다)
  2. 게시물 설명(여기서 태그해도 됩니다)
  3. 청소년에게 유해하진 않은 내용인지 아닌지 체크
- 올리기를 했을 때, 만약 너무 유해하거나 문제가 있는 게시물이면 AI가 올리기 처리하는 과정에서 거부합니다.

게시물을 확대했을 때 화면
- 왼쪽에 > 모양의 화살표가 있고 그걸 열면 리코딩 과정에서 얻어온 설계도로 유사한 옷을 추천해줍니다. (상의, 하의, 신발로 해서 3X3의 정사각형 9개로 해서 말이죠. 이때 뒤는 약간의 흐림처리가 됩니다. 빈공간을 클릭하면 옷 화면이 내려가고 게시물 화면으로 돌아옵니다.)
- 우측에 위에서부터 순서대로 별점, 댓글, 공유가 뜹니다.
- 여기선 올린 3개의 이미지를 넘기면서 볼 수 있습니다.

### 4.1.1 PC Web Studio Spec (Roblox-style)
- **Environment**: **PC Web Only** (모바일 앱 지원 불가).
- **Base Asset**: 시스템이 제공하는 '기본 가면(Base Mask)' 또는 유저가 이전에 만든 내 가면을 불러옴.

- **Editing Rules**:
  - **Exterior Only**: 가면의 **바깥쪽(Exterior)** 표면에만 블록/재질 부착 가능 (안쪽 얼굴 닿는 면 불가).
  - **No Floating**: 모든 파츠는 가면 표면 또는 다른 파츠에 **물리적으로 연결(Attached)**되어야 함 (공중 부양 파츠 저장 불가).
  - **Center Lock**: 작업 화면의 중앙은 항상 가면의 중심축(Center)에 고정됨.

- **Block System**:
  - 다양한 재질(Material), 모양(Shape), 크기(Scale)를 자유자재로 변형 가능한 블록 시스템 제공.

- **Sync & Registration**:
  - **Save**: 작업물은 PC 로컬 저장소 및 서버 '저장고(Vault)'에 코드로 저장됨.
  - **Link to App**: 웹/앱에서 **[유저 코드 + 이메일 + 비밀번호]** 입력 시, 해당 가면 데이터가 마이페이지와 연동(Import)되어 상점 등록이 가능해짐.
