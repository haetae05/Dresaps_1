## 1. AI Justice System (AI 판결 시스템)

**From Domain 01**: "인간보다 공정한, 감정 없는 심판."

### 1.1 Evidence Freezing (증거 보존)
- **Problem**: 신고자가 악의적으로 신고 후 취소하거나, 피신고자가 증거 인멸을 위해 게시물을 삭제하는 경우.
- **Solution (Infrastructure)**:
    - 신고(Report) 접수 즉시, 해당 게시물의 이미지를 **[Domain 09]**의 스냅샷 기술을 활용하여 `evidence_archive` 버킷에 **Deep Copy**한다.
    - 이 데이터는 판결이 확정될 때까지(최대 30일) 어떤 권한으로도 삭제가 불가능하다.

### 1.2 Cooling-off Period (숙려 기간)
- **Logic**: 모든 신고는 접수 즉시 처리되지 않고, **24시간의 대기 시간**을 가진다.
    - **Purpose**: 유저 간의 일시적 감정 싸움이나 보복성 '맞신고'가 AI 심판대에 오르는 것을 방지.
- **Trigger**: `processed_at` (Current + 24h) 도래 시 Cloud Functions 스케줄러가 AI 심사 로직 호출.
- **Safety Net**: 단, `NUDITY` (심각한 유해성) 신고가 **1시간 내 3회** 누적 시, 24시간 대기 없이 **즉시 블라인드(Blur)** 처리 후 심사.

### 1.3 Verdict Logic (판결)
- **Engine**: Google Cloud Vision API (Image) + Perspective API (Text).
- **Outcome**:
    - **Guilty (유죄)**:
        - 해당 게시물 **Soft Delete** (본인에게만 보임, 타인에게는 '삭제된 게시물' 표시).
        - **Reputation Score** 자동 하락 (-5점).
        - 경고 알림 발송 ("커뮤니티 가이드를 위반하여 게시물이 조치되었습니다.").
    - **Innocent (무죄)**:
        - 신고자에게 "위반 사항 없음" 통보.
        - 동일 유저가 허위 신고 3회 누적 시 **역제재(Penalty)** 적용.

### 1.4 Human Appeal (이의 제기)
- **Condition**: AI가 '유죄' 판결을 내린 경우에만 활성화 버튼 노출.
- **Process**: 유저 소명 작성 -> **GM Libe**의 'Operator Desk'로 전달.
- **Tone**: 복구 승인 시 "AI가 실수했네! 미안, 내가 사과할게. 필름 3장 받아." (Humorous & Generous).

## 2. Reputation System (평판 및 제재 - From Domain 02)

**"점수는 스스로 지키는 것. 한 번 내려가면 올리기 힘들다."**

| 점수 구간 | 상태 (Status) | 제재 및 제한 (Restrictions) |
| :--- | :--- | :--- |
| **90.0 ~ 100** | **Normal** | 모든 서비스 정상 이용 가능. |
| **80.0 ~ 89.9** | **Warning** | **경제 활동 제한 (D07/D08)**:<br>- 상점(Shop) 물품 등록 불가.<br>- 패셔니스타(Fashionista) 신청 불가 (기존 자격 박탈). |
| **75.0 ~ 79.9** | **Danger** | **사회 활동 제한 (D06)**:<br>- 스타일 컵(Style Cup) 매칭 풀에서 제외 (Shadow Ban).<br>- 댓글 작성 및 채팅 차단(Mute). |
| **0.0 ~ 74.9** | **Critical** | **서비스 이용 제한 (Hell Gate)**:<br>- 로그인 시 '정화 교육 캠페인' 팝업 강제 노출 (Skip 불가).<br>- **API 요청 속도 강제 지연 (Latency Penalty 3000ms).** |

*(Note: 별도의 점수 회복 시스템은 존재하지 않음. 유저는 성실한 활동으로 신고를 당하지 않는 것만이 유일한 생존 전략임.)*

## 3. Abuse Protection (어뷰징 방어 - From Domain 05)

### 3.1 Rating Shield (별점 방어)
- **Rate Limiting**: 클라이언트가 아닌 서버(Redis) 레벨에서, 동일 유저가 **3초 이내** 재평가 API를 호출하면 `429 Too Many Requests` 에러 반환.
- **Hater Logic**:
    - 최근 20개 평가의 평균 점수가 **1.5점 미만**인 유저 식별.
    - 해당 유저의 `Rating Weight`를 **1.0 -> 0.1**로 강제 하향 조정하여 타인의 평점에 미치는 영향력 무력화.

### 3.2 Ghost Protocol (Shadow Ban Tech)
- **Target**: 악성 유저(Hater, Spammer) 및 평판 점수 `Danger` 등급 이하.
- **Mechanism**:
    - 유저는 본인이 차단당했다는 사실을 모른다. (댓글 작성 시 본인 화면에는 정상적으로 보임).
    - 하지만 실제 DB에는 저장되지 않거나 `is_hidden = true` 상태로 저장되어 타인에게는 보이지 않는다.
    - **Effect**: "왜 반응이 없지?" 하고 스스로 지쳐서 나가떨어지게 만듦.

## 4. Work-Life Balance (운영자 보호)

**"GM도 사람이다."**

### 4.1 Auto-response (문의 자동화)
- **Smart Inquiry**: 이메일 문의 버튼 클릭 시 앱 실행 전 **FAQ 팝업** 강제 노출 ("필름이 안 들어왔나요?", "결제가 안 되나요?").
- **Context Injection**: 이메일 클라이언트 실행 시 본문에 `User ID`, `Device Model`, `OS Version`, `Last Error Log Code`를 자동 삽입하여 불필요한 핑퐁 방지.

### 4.2 Operator's Desk
- **To-Do List**: GM은 하루에 한 번, 시스템이 필터링 한 '**이의 제기(Appeal)**' 건만 처리하면 된다.
- **No Chat**: 실시간 상담 채팅(Live Chat)은 운영하지 않는다. 모든 문의는 비동기(Async) 이메일 원칙.

## 5. Data Privacy & Compliance (규제 대응 - From Domain 14)

### 5.1 Right to Erasure (잊혀질 권리)
- **Hard Delete**: 유저가 계정 삭제(회원 탈퇴) 요청 시.
- **Scope**:
    - **즉시 삭제**: 프로필 정보, 팔로우/팔로잉 관계, 원본 신체 사진(Raw Data).
    - **보관(Retention)**: 결제 내역(1년 - 세법), 사기/제재 기록(1년 - 재가입 방지 해시).

### 5.2 GDPR & COPPA
- **Age Gate**: 가입 시 생년월일 입력 후 14세 미만 식별 시 가입 절차 강제 종료 및 입력 데이터 즉시 파기.
- **Consent**: EU/EEA 지역 접속 시 CMP(Consent Management Platform) 팝업을 통해 쿠키 및 데이터 수집 동의 명시적 획득.
